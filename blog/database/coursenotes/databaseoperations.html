Character encoding: The computer saves data in binary mode, and the characters entered by the user are converted to binary according to certain rules and then saved. # Tip

Data type:

  1. Round numbers tinyint,smallint,mediumint,int,bigint
  2. Floating point number Used to store decimals. float(4 bytes),double(8 bytes)
  3. Number of fixed points The decimal used to hold the ** exact precision **. decimal,numberic
  4. The character string type char,varchar,tinyblob,blob,mediumblob longblob,tinytext,text,mediumtext,longtext
  5. Date and time year,date,time,datetime,timestamp

** Floating-point number types are not very accurate ** Floating-point number types are suitable for storing decimals as approximations rather than as exact values. FLOAT numbers, ** integers and decimals with more than 6 digits ** may lose precision. DOUBLE numbers, ** integers and decimals with more than 15 digits ** may lose accuracy. Floating-point numbers can also lose precision in mathematical calculations.

DECIMAL(M,D) M: The total length of the integer part plus the decimal part. The value ranges from 0 to 65. The default value is 10. The value ranges from 0 to 30. The default value is 0. The value must be D<=M.

varchar Variable length, char fixed length

blob is commonly used to store binary data such as images and pdf documents

** The single row length of a data table in MySQL is limited to 65 535 bytes. **. The single-line length limit means that the total number of fields of common data types other than the BLOB and TEXT series in a record cannot exceed the limit. **BLOB series and TEXT series fields are not limited by the length of a single line ** and only take up 9 to 12 bytes of storage space of a single line.

When using the YEAR type, be sure to distinguish between the character '0' and the number 0. The former represents the year 2000 and the latter represents the year 0000

Differences between TIMESTAMP types and DATATIME types: The TIMESTAMP type has a smaller value range than the DATATIME type. The value of the TIMESTAMP type depends on the time zone, and the same date and time taken from two different time zones may display different results. If the date and time are inserted, the system will convert the date and time according to the time zone set by the current system and save them. When the date and time are obtained, the data is converted to the corresponding time zone and displayed.

# The only constraint

A unique constraint is used to ensure the uniqueness of the values in a field. After a unique constraint is set, the values in the field cannot be repeated. UNIQUE constraint With the unique keyword setting, you can set one or more unique constraints in a data table. Unique constraints can be set either when the data table is created or when the data table is modified.

# Tip

There are two ways to set unique constraints when creating a data table: ** column-level ** constraints and ** table-level ** constraints. The difference between column-level constraints and table-level constraints: ** Column-level ** Constraints are defined in the column, immediately after the data type of the field, ** only applies constraints to that field **. ** Table-level ** constraints are field-independent and can ** constrain single or multiple fields of a data table **. When a table level unique constraint is built on multiple fields, uniqueness is ensured through multiple fields, and as long as one of the fields is different, the result is unique. When a table-level unique constraint is built on only one field, it has the same effect as a column-level constraint.

# primary key constraint

The primary key constraint is used to set one or more fields as the primary key of a table. A primary key constraint is a combination of a non-null constraint and a unique constraint. The constrained field cannot have duplicate values or NULL values. PRIMARY KEY constraint Only one primary key constraint can be set in each data table. The primary key constraint can be set when the data table is created or when the data table is modified.

You can set column-level or table-level primary key constraints when creating a data table. Column-level primary key constraints can be set only for a single field. Table primary key constraint refers to the setting of a single field or multiple fields. When the primary key constraint is set for multiple fields, a composite primary key is formed.

# Tip

Precautions for using AUTO_INCREMENT: Only one field in a data table can be set AUTO_INCREMENT. The data type of the set AUTO_INCREMENT field must be an integer, and the field must be set with a unique constraint or a primary key constraint. If a NULL value is inserted for an auto-grow field, or if the auto-grow field is omitted when data is inserted, the auto-grow value is used for the field. If a specific value is inserted, the value is not automatically increased. By default, the AUTO_INCREMENT field is set to increment from 1. If a specific value is inserted that is greater than the auto-grow value, the next auto-grow value is the maximum value in the field plus 1. When DELETE is used to delete data, the automatic growth value does not decrease or fill the gap.
