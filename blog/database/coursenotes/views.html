# View

Definition: View is a virtual existence of a table, the view of the data from the database data tables, these data tables are called ** basic table **. Through the view, you can not only see the data in the basic table, but also add, modify and delete the data in the basic table. Only the SQL statements ** that define the view are saved in the database, and the data corresponding to the view is not saved. When * * * * * * in the data base table change * * * *, * * * * view query from a * * * * data also does change * *, * * * * * * view when through modifying data * *, * * * * * * in the data base table is * * * * * * change will happen.

Advantages: It can not only simplify the user's understanding of the data, but also simplify the operation of the data and so on.

# Single table based view

# Create


CREATE [OR REPLACE]
[ALGORITHM = {MERGE | TEMPTABLE | UNDEFINED}]
[DEFINER = user]
[SQL SECURITY { DEFINER | INVOKER }]
VIEW View name [(field list)] AS select_statement;
[WITH [CASCADED | LOCAL] CHECK OPTION];


-- OR REPLACE: Optional. If a view with this name already exists in the database, the view is replaced. If no view exists, the view is created.
- ALGORITHM clause: Used to declare how MySQL handles views, MERGE statements that reference views and statements that define views, TEMPTABLE saves the results of views to temporary tables, UNDEFINED means no processing is specified (MERGE is used by default).
-- DEFINER clause and SQL SECURITY clause: Used to declare access rights when executing a view, the user value in the DEFINER clause is usually "' username '@' host address '", CURRENT_USER, or CURRENT_USER(). If the DEFINER clause is omitted, the default executor is the user who created the view.
View name: Indicates the name of the view to be created. The name must be unique in the database and cannot be the same as another data table or view.
-- select_statement: specifies a complete SELECT statement that detects records that meet select_statement conditions from a data table or view and imports these records into a view.
-- WITH CHECK OPTION: indicates the check condition for view data operation. If it is omitted, the check is not performed. Its value can be CASCADED (default), and all conditions defined about views and tables are satisfied during data operation. LOCAL: Only the conditions defined in the view must be met when you operate data.

# multi-table based view

# Create

Creating a view based on two or more tables is called a multi-table based view. When creating a VIEW based on multiple tables, you only need to specify the SELECT statement in the CREATE VIEW statement as the SQL statement for multi-table query.

# View view

To fully understand a view, you need to view the view's fields, status, and creation statements

# View the field information of the view

-- Short for
