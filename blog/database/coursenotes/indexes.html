# Index

An index is a separate, physical storage structure that can be created for a data table to quickly find the target data and to sort the values in one or more columns of the data table.

# Tip

Common indexes: Common, unique, primary key, Full text, Space A common INDEX is a basic index type in MySQL that is defined using the KEY or index keyword and does not require any restrictions.

A unique index is used to limit the number of duplicate values in a field. The unique index is automatically created when a unique constraint is created. Fields that create unique indexes are allowed to have NULL values, and NULL values are not treated as duplicate values.

The primary key index identifies each record based on the uniqueness of the primary key itself. When a primary key constraint is created, the primary key index is automatically created. The field of the primary key index cannot have NULL values.

Full-text index is mainly used to improve the query efficiency of large data fields. Full-text indexes can only be created on fields of type CHAR, VARCHAR, or TEXT.

Spatial indexes can only be created on fields of spatial data types, which store spatial data that contains location, size, shape, and its own distribution characteristics. To create a field for a spatial index, you must declare it NOT NULL.

The categories are based on the number of fields to be indexed: Single-column index Composite index

A single-column index creates an index on a single field in a table. It can be a common index, a unique index, or a full-text index, as long as the index corresponds to only one field in the table.

Composite index refers to the creation of indexes on multiple fields. The setting of fields follows the "leftmost prefix" principle. The most frequently used field is placed on the leftmost of the index field list. The index is used when the first of these fields is used in the query criteria.

Although the index can improve the query speed of the data, but at the same time the index will occupy a certain amount of disk space **, ** and ** in the ** creation and maintenance of the index **, its consumption of ** time is increased with the increase in the amount of data. Therefore, it is necessary to consider the advantages and disadvantages of indexes when using indexes.

Create a transaction for a data table to ensure the integrity of a set of SQL statements manipulating the data. Create ** views ** for data tables to improve data security. Build ** index ** for data table to speed up the query efficiency of data.

# Use of index

# Create

# The index is created while the table is being recreated

CREATE TABLE Table name (
Field name Data type [Field properties],
...
{INDEX|KEY} [index name] [index type] (Field list) {index | key}
| UNIQUE [INDEX|KEY] [index name] [index type] (Field list)
| PRIMARY KEY [index type] (Field list)
| {FULLTEXT|SPATIAL} [INDEX|KEY] [index name] (Field list)
);

-- {INDEX|KEY} : INDEX and KEY are synonyms, indicating the index.
- Index name: Specifies the name defined for the index to be created. If this option is not used, single-column indexes use the field where the index is created by default. Compound indexes use the name of the first field by default.
Index type: Optional, different storage engines allow to specify the index type when creating an index, the syntax is USING {BTREE | HASH}, for example InnoDB and MyISAM support BTREE, MEMORY supports BTREE and HASH.
-- UNIQUE: indicates a unique index.
-- FULLTEXT: indicates the full-text index.
-- SPATIAL: indicates the spatial index.

# View the index

-- You can also use the SHOW CREATE TABLE statement to view the index information while viewing the table creation sentence

SHOW {INDEXES|INDEX|KEYS} FROM table names;

-- View the meaning of each field in the index's results:

-- Table Indicates the name of the data table where the index resides
-- Whether the Non_unique index can be repeated, 0 can not, 1 can
-- Key_name Indicates the name of the index. If the index is a PRIMARY key index, the name is primary
-- Seq_in_index Specifies the index number. The default value starts from 1
-- Column_name Specifies the column for establishing an index
-- Collation Indicates whether the index field is sorted. A indicates whether the index field is sorted. NULL indicates whether the index field is sorted
Cardinality Indicates the possibility of using indexes during MySQL connections. A larger value indicates a higher probability
-- Sub_part Length of the prefix index. If all field values are indexed, Sub_part is NULL
How is keyword Packed, if not packed to NULL
-- Null Indicates whether the index field contains NULL values
- the Index type Index, optional value is: FULLTEXT, HASH, BTREE, RTREE
-- Comment Indicates comment information about the index field
-- Index comment Indicates the comment added when an index is created
-- Visible Whether the index is visible to the query optimizer
-- Expression What expression is used as the index field

# Analyzes whether SQL uses indexes

# Delete the index


ALTER TABLE Table name DROP INDEX Index name;
or
DROP INDEX Index name ON table name.


When deleting a PRIMARY key index, the index name of the PRIMARY key index should be primary, which is included in backquotes ('')
# Syntax 1
ALTER TABLE Table name DROP INDEX 'PRIMARY';
# Syntax 2
DROP INDEX 'PRIMARY' ON Table name;
